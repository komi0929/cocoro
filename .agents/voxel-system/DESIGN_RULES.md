## 1. 設計哲学

### 1.1 品質 ≠ ボクセル数
ボクセルを増やしても品質は上がらない。品質を決めるのは：
- **プロポーション** — 頭:体の比率
- **色の選択** — 暖色系、少ない色数で調和
- **パーツ配置の意図** — 1ドットに意味を持たせる
- **シルエットの可愛さ** — 角丸、丸みを帯びたフォルム

### 1.2 ブランドの統一感
すべてのアセット（アバター/家具/環境）が「同じ世界の住人」に見えること:
- **同じボクセル粒度** — 1ブロックの相対サイズが統一されている
- **同じカラーテイスト** — 暖色・パステル基調
- **同じ角丸ルール** — cornerR=1（1ブロック削り）
- **同じライティング** — 上からの暖色光、柔らかい影

### 1.3 抽象化原則
参考画像は「具体例」であり「テンプレート」ではない。
新しい動物・家具・環境を作るとき、以下を抽象的に適用する：
- クマの茶色 → 「その動物固有の暖色メインカラー」
- ペンギンの紺 → 「その動物固有のクールカラー（例外もOK）」
- ツリーハウスの木目 → 「そのテーマ固有のテクスチャパターン」

## 2. アバター設計原則（NORTH_STAR_2 分析）

### 2.1 プロポーション（絶対ルール）

```
        ┌─────────┐
        │  耳     │  耳高: headH × 0.25
        ├─────────┤
        │         │
        │   頭    │  headH ≈ bodyH × 1.1〜1.3（大頭身）
        │  ○  ○  │  headW ≈ bodyW × 1.0〜1.2
        │   ▽    │
        ├─────────┤
        │   体    │  
        │ ┃    ┃ │  armH ≈ bodyH × 0.7
        │ ┃    ┃ │  armW = 2〜3ブロック
        ├─┘    └─┤
        │ ╻    ╻ │  legH ≈ bodyH × 0.35〜0.5
        └─┘    └─┘  legW = 3〜4ブロック
```

### 2.2 顔のルール

| 要素 | ルール | 理由 |
|------|--------|------|
| 目 | **3×3 or 4×3 黒** + 白ハイライト1〜2ドット（左上） | 参考全キャラ共通 |
| 目の間隔 | headW × 0.35 〜 0.45 の間に収まる | 近すぎ/遠すぎ防止 |
| 鼻 | 2×2 以下、色=ピンクor黒or動物固有色 | 小さめが可愛い |
| 口 | 1ドット高さ×3〜4幅、頭の下25%位置 | 控えめ |
| ほっぺ | **ピンク系1〜2ドット**、目の下外側 | kawaii必須（全キャラ） |
| マズル | 白/明色の楕円を正面に塗る、突出MAX 1ブロック | フラット＆かわいい |

### 2.3 色の原則

| ルール | 詳細 |
|--------|------|
| メインカラー | 動物固有の暖色 (例: クマ=#A07030, ネコ=#9E9E9E) |
| シャドウ | メインの 75% 明度 |
| ハイライト | メインの 115% 明度 |
| ベリー（お腹） | 白〜クリーム系 |
| アクセント | 耳内側=ピンク、アイテム=固有色 |
| 色数上限 | 1キャラ **5色以内**（メイン+シャドウ+ハイライト+ベリー+アクセント） |

### 2.4 新動物追加時のチェックリスト

新しい動物（例: フクロウ）を追加するとき:
1. [ ] その動物の特徴的シルエットを角丸ブロックで表現できるか？
2. [ ] headW:bodyW が 1.0〜1.2 の範囲内か？
3. [ ] 目が顔面積の 20-30% か？
4. [ ] ほっぺドットがあるか？
5. [ ] 色が5色以内か？
6. [ ] 手持ちアイテムがあるか？
7. [ ] 既存8体と並べて「同じ世界の住人」に見えるか？

## 3. 家具・小物設計原則（NORTH_STAR_3 分析）

### 3.1 参考画像から読み取れるルール

参考画像に含まれるアイテム分析:
- **ソファ**: ベージュ基調、3ブロック高背もたれ、木脚4本、クッション柄あり
- **テレビ**: レトロブラウン木目台、アンテナ付き、画面は暗色平面
- **冷蔵庫**: クリーム色、取っ手=銀2ドット、上下分割
- **椅子**: ウッド系、背もたれ板+座面+4脚
- **ランプ**: 木スタンド+クリーム四角シェード
- **時計**: 丸型ピクセルアート（壁掛け）
- **花瓶+花**: 白い花瓶＋赤い花＋緑の葉
- **植木鉢**: テラコッタ色+緑のサボテン/多肉
- **本**: 積み上げ3冊、各色違い
- **毛糸玉**: ターコイズ、溝パターン
- **コーヒーカップ**: 小さな白カップ+ソーサー
- **イーゼル+絵**: 木フレーム+風景画（ピクセルアート内のピクセルアート）
- **レコード/焚き火跡**: 小さな床置き

### 3.2 抽象的ルール

| 要素 | ルール |
|------|--------|
| スケール | アバターの **1/2〜1/3** サイズ |
| 角丸 | cornerR=1（全方向） |
| 木材 | 暖色系ブラウン2色（明+暗）、水平ストライプで木目表現 |
| 金属 | ライトグレー+ダークグレー、ハイライト1ドット |
| 布 | パステル/ナチュラル色、クッション柄=格子or水玉 |
| ガラス | 半透明ライトブルー or 暗色反射 |
| 接地点 | 脚は必ず **少なくとも2ブロック離れた4点** or **面接地** |
| ディテール | 取っ手、ノブ、ステッチなど **1〜2ドットの小さなアクセント** |
| 生活感 | 本は積み上げ、カップはソーサー付き、花は花瓶に |

### 3.3 新家具追加時のチェックリスト
1. [ ] アバターと同じ世界に見えるスケールか？
2. [ ] 暖色系カラーパレットか？
3. [ ] 角丸 cornerR=1 が適用されているか？
4. [ ] 1〜2ドットのディテールアクセントがあるか？
5. [ ] 参考画像の家具と並べて違和感がないか？

## 4. 環境・テーマ設計原則（NORTH_STAR_1 + NORTH_STAR_4 分析）

### 4.1 ポリゴン粒度の基準

NORTH_STAR_1（ツリーハウス）から読み取れる粒度:
- **壁面**: 木目テクスチャは **1ブロック=色の変化単位** として細かく表現
- **床**: 木の板が **2〜3ブロック幅** で交互色
- **窓**: 丸窓のフレームが **4〜5ブロック径** で表現
- **苔/ツタ**: 壁面に **1〜2ドットのランダム散布**
- **小物**: 棚の上に **4〜6ブロック** のミニアイテム

### 4.2 空間密度（NORTH_STAR_4 から）

NORTH_STAR_4（ペンギンの部屋）の空間構成:
- **壁面**: 絵画(大)＋棚(3段)＋ドア＋壁の色分け（上=明/下=暗トーン）
- **床面**: 木目パターン、アイテムが30-40%を占める
- **小物散布**: ブロック、本、毛糸玉、ポーション瓶など **10個以上の小物が床に散在**
- **高さの使い方**: 棚は壁の中段〜上段に配置、照明は天井中央

### 4.3 8テーマへの適用（抽象的）

各テーマは **世界観は異なるが粒度・品質は同じ**:

| テーマ | 世界観のキーワード | カラーパレット方向 |
|--------|-------------------|-------------------|
| 地下室 | 鍾乳洞+クリスタル+パイプ | 紫+ティール+グレー |
| ロフト | 暖炉+本棚+窓 | ブラウン+クリーム+赤 |
| ツリーハウス | 木+葉+キノコ+ツタ | ブラウン+グリーン+黄 |
| ビーチ | 砂+ヤシの木+波+貝殻 | サンド+ターコイズ+白 |
| 屋上 | ビル群+アンテナ+ライト | グレー+ネオン+夜空色 |
| 宇宙 | 金属パネル+ホログラム+星 | ダークブルー+シアン+銀 |
| 深海 | サンゴ+クラゲ+海藻 | ディープブルー+ピンク+緑 |
| 火山 | 溶岩+黒曜石+蒸気 | 黒+赤+オレンジ |

### 4.4 環境装飾の品質ルール
1. **壁面は情報密度を上げる** — 絵画、棚、ポスター、窓、配管など
2. **床に小物を散らす** — 本、ブロック、カップ、花など生活感
3. **高さを使う** — 棚、天井照明、つり下げ装飾
4. **1ブロック=1テクスチャ単位** — 木目、金属パネル等を適切に表現
5. **暖色照明** — 天井中央からの暖かい光が基本

## 5. システムアーキテクチャ

### 5.1 ファイル構成

```
src/components/three/voxel/
├── VoxelGrid.tsx        — レンダラー（InstancedMesh + AO + 面カリング）
├── VoxelPrimitives.ts   — 描画プリミティブ（fillRoundedBox3D, drawEye等）
├── VoxelDesign.ts       — デザインシステム（パレット、kawaii原則）
├── VoxelEngine.ts       — 設定駆動エンジン（JSON→VoxelData）
├── VoxelCatalog.ts      — テンプレートカタログ
├── VoxelAvatars.ts      — アバター生成（8種+拡張可能）
├── VoxelFurniture.ts    — 家具生成
├── VoxelEnvironments.ts — 環境要素生成
├── VoxelModels.tsx      — プロシージャルモデル
├── VoxelAssets.tsx       — Reactコンポーネントラッパー
└── VoxelAudit.tsx       — 品質監査UI（?audit=voxel）
```

### 5.2 新アセット追加フロー

1. ユーザーが「Xを作れ」と指示
2. このファイル（`DESIGN_RULES.md`）を読む
3. NORTH_STAR画像を参照して品質基準を確認

5. `VoxelAudit.tsx` に登録
6. ブラウザで `?audit=voxel` を開き視覚確認
